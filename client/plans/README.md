# client/plans/

Implementation Plans (HOW) scoped to **`client/` only** — written by
`implementation-planner`, consumed by `implementer`.

A plan lands here when the change touches only this package; a plan spanning
several packages goes to the **top-level `plans/`** instead. Pairs with
`client/specs/` (the WHAT/WHY). One flat `<slug>.md` per feature, written after
the developer approves. See `.claude/agents/implementation-planner.md`.
